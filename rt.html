Summer 2022 Internship at Ready Tensor

Creating a library of deployable Machine Learning Models for Ready Tensor Inc.

Background on Ready Tensor

Ready Tensor Inc. is a machine learning (ML) startup that provides a platform for users to create, edit, and share different machine learning models. Users may also download these models which are containerized using Docker which makes them deployable in production settings. More information about the company can be found here.

My Contribution

During my internship at Ready Tensor, I was responsible for creating a library of containerized machine learning models for two catrgories: binary classification and text classification. To be precise, a binary classification algorithm determines whether or not an observation belongs in either one of two classes (i.e. if a cell phone is either an Apple or an Android based on some features of the phone). A text classification algorithm is simply the categorization of a corpus of text into a specific category (i.e. would this review be considered a good, bad, or neutral review). The purpose of this library is to provide contributing users with a baseline benchmark by which to measure their own models against.

I then was tasked with researching the different types of algorithms for each category I was assigned. Listed below are some of the algorithms and tools used to create the library of machine learning models

Algorithms

  • Random Forest Classifier
  • Logistic Regression Classifier
  • K-Nearest Neighbors Classifier
  • Support Vector Classifier
  • Recurrent Neural Networks
  • Convolutional Neural Networks
  • Naive Bayes Classifier

Tools and Packages

  • TensorFlow
  • Pytorch
  • SciKitLearn
  • Pandas
  • NumPy
  • Docker
  • Lime
  • Shapely
Each algorithm I created had to fit within the specified Ready Tensor framework. This included providing a data cleaning pipeline, hyperparameter tuning, and a model evaluation output in the Docker container. So I used a variety of datasets to ensure that the cleaning pipeline was working as intended as well as evaluate how well the model would perform. If no issues arose, these algorithms would then be uploaded to the platform.

Final Result

After 3 months of creating and refining ML models, the two other interns and I were able to create a library of over 50 industry deployable ML models.

Future Expansions

As a whole, the project was a success and provided a great starting point for future iterations and improvements. One big improvement we would like to see made going forward would be the automating of the data loading and transformation process. For the sake of this project and creating a feasible application, we exported, cleaned, and visualized the data manually and stored them in Google Drive folders. Having these processes automated would save much time on our end and would allow for a more continuous upload of current health data.
